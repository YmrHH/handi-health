# bigScene（寒岐智护·慢性病随访健康预警管理平台）

## 启动

在 `e:\智能康养系统\Complete system\test(vue)\bigScene` 目录执行：

```bash
npm install
npm run dev -- --host
```

默认访问地址：`http://localhost:5190/`

## 数据与接口策略

### 1) 后端地址

- 大屏端统一通过 `src/api.ts` 的 axios 实例请求：
  - `baseURL: http://192.140.173.165:8081`

### 2) 鉴权头策略

- 仅当本域 `sessionStorage` 中存在真实值时才附带请求头：
  - `token` 存在时才带 `Authorization: Bearer <token>`
  - `userId` 或 `xUserId` 存在时才带 `X-User-Id`
- 不再使用任何假 token / 假 userId 兜底。

### 3) 缓存与降级

- `src/api.ts` 保留内存 TTL 缓存，减少重复请求抖动。
- 页面展示层保留结构化降级：接口字段缺失时以可解释的回退数据维持页面完整，不破坏主布局。

### 4) 关键接口约束

- `fetchPatientSummary(pageSize)` 使用后端允许范围，当前默认与主要调用均为 `200`。
- 首页榜单使用“直接统计优先 + 后台增强补齐”策略，避免阻塞首屏。

## 验收清单（建议每次发版前执行）

- 页面级：
  - 五个舱均为单屏展示，无纵向滚动。
  - 浏览器常见缩放比例下不出现卡片遮挡、裁切、重叠。
  - 底部状态带固定贴底且不遮挡趋势区。
- 数据级：
  - 首页左侧病种排行、医生负载可稳定出数。
  - 干预服务舱图表、指标、事件流来自真实接口聚合。
  - 告警联动舱与风险画像舱在激活切回时可刷新数据。
- 工程级：
  - `npm run build` 通过。
  - 主要页面无新增 linter 报错。

# 慢病随访小程序 - SSM 后端接口文档（最终版）

> 对应前端代码：`config/api.js`、`utils/request.js`、`api/*.js`
>
> **统一前缀**：`/api`（由前端 `API_PREFIX` 决定）

---

## 1. 通用规范

### 1.1 BaseURL

- 开发示例：`http://localhost:8080`
- 生产示例：`https://your-domain.com`

前端最终请求：

```
{API_BASE_URL}{API_PREFIX}{url}
```

示例：
- `POST https://your-domain.com/api/auth/login`

### 1.2 鉴权

推荐 JWT Bearer Token：

```
Authorization: Bearer <token>
```

- 未登录/Token过期：建议返回 HTTP 401（或业务码 `code=401`）
- 角色不匹配（例如 staff 调 patient 接口）：建议返回 HTTP 403（或业务码 `code=403`）

> 前端会在 `utils/request.js` 自动附加 token。

### 1.3 统一响应结构

建议所有接口返回：

```json
{
  "code": 0,
  "msg": "ok",
  "data": {}
}
```

- `code = 0`：成功
- `code != 0`：失败（`msg` 给出可读错误信息）

> 前端兼容：
> - `code/msg/data`
> - 或 `{ success: true/false, message, data }`

### 1.4 通用错误码建议

| code | 含义 | 备注 |
|---:|---|---|
| 0 | 成功 | - |
| 400 | 参数错误 | 缺字段/格式不对 |
| 401 | 未登录/Token过期 | 需重新登录 |
| 403 | 无权限 | 角色不匹配 |
| 404 | 资源不存在 | id 不存在 |
| 409 | 冲突 | 重复提交/状态冲突 |
| 500 | 服务端异常 | - |

### 1.5 分页（如需要）

- 推荐参数：`page`（从1开始）、`pageSize`
- 推荐返回：`{ list, page, pageSize, total }`

## 3. 文件与通用能力

### 3.1 通用文件上传

- **URL**：`POST /files/upload`
- **鉴权**：是
- **Content-Type**：`multipart/form-data`

**FormData：**
| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| file | File | 是 | 图片/语音等 |
| bizType | string | 否 | `tongue` / `avatar` / `feedback` / `voice` 等 |
| patientId | string | 否 | 可选（也可从 token 推导） |
| taskId | string | 否 | 随访任务 id |
| alertId | string | 否 | 告警 id |

**Response(data)：**
```json
{
  "fileId": "f_90001",
  "url": "https://your-cdn.com/files/xxx.jpg",
  "name": "tongue.jpg",
  "size": 123456,
  "mime": "image/jpeg"
}
```

> 前端：
> - 患者舌象：会展示 `url` 并缓存上传时间
> - 医护随访：会把 `fileId/url` 放进随访记录提交

### 3.2 获取服务器时间（可选）

- **URL**：`GET /common/server-time`
- **鉴权**：否

**Response(data)：**
```json
{
  "serverTime": "2026-01-14 10:20:30"
}
```

## 7. 联调说明（给后端同学）

1. 先实现：`/auth/login`、`/patient/home`、`/staff/tasks`、`/staff/alerts`，即可跑通最小闭环。
2. 返回结构务必统一：`{ code, msg, data }`。
3. 微信小程序：需要配置 request / uploadFile 合法域名，生产环境建议 HTTPS。
4. 图片/文件 url 必须可公网访问，否则前端预览会失败。

```json
}
```

> 说明：后端收到该请求后，应在 `patient_messages`（或消息表）中新增一条消息记录，患者端通过 `GET /patient/messages` 可查看。

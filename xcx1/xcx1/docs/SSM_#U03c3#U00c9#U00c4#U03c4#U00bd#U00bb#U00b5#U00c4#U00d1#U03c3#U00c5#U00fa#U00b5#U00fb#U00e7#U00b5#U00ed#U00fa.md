> 适用：本项目 uni-app（微信小程序端）对接 SSM（Spring + SpringMVC + MyBatis）后端。
>
> 本文档与前端代码 **1:1 对应**：`config/api.js`、`utils/request.js`、`api/*.js`。
>
> **统一前缀**：`/api`

## 1. 约定与通用规范

### 1.1 BaseURL

- 开发：`http://localhost:8080`（示例）
- 生产：`https://your-domain.com`

前端会拼接：`{API_BASE_URL}{API_PREFIX}{url}`
- `API_PREFIX` 默认为 `/api`

例如：
- 登录：`POST https://your-domain.com/api/auth/login`

### 1.2 鉴权

- 推荐：JWT Token（Bearer）
- 前端会自动携带：

```
Authorization: Bearer <token>
```

未登录或 token 失效：
- HTTP：建议返回 `401`
- 或业务码：`code=401`

### 1.3 统一响应结构

建议所有接口返回：

```json
{
  "code": 0,
  "msg": "ok",
  "data": {}
}
```

- `code = 0`：成功
- `code != 0`：失败（`msg` 给出可读错误信息）

> 前端兼容两种：
> - `code/msg/data`
> - 或 `{ success: true/false, message, data }`

### 1.4 通用错误码建议

| code | 含义 | 说明 |
|---:|---|---|
| 0 | 成功 | - |
| 400 | 参数错误 | 缺字段/格式不对 |
| 401 | 未登录/Token过期 | 需重新登录 |
| 403 | 无权限 | 角色不匹配 |
| 404 | 资源不存在 | id 不存在 |
| 409 | 冲突 | 重复提交/状态冲突 |
| 500 | 服务端异常 | - |

### 1.5 时间字段

- 推荐统一用：`yyyy-MM-dd` 或 `yyyy-MM-dd HH:mm:ss`
- 前端：
  - 日期（如随访日期、今日日期）常用 `yyyy-MM-dd`

## 2. 认证与账号

### 2.1 登录

- **URL**：`POST /auth/login`
- **是否鉴权**：否

**Request(JSON)：**
```json
{
  "role": "patient",
  "username": "13800000000",
  "password": "123456"
}
```

- `role`：`patient` / `staff`

**Response：**
```json
{
  "code": 0,
  "msg": "ok",
  "data": {
    "token": "xxx",
    "user": {
      "id": "u_10001",
      "role": "patient",
      "name": "张阿姨",
      "phone": "13800000000",
      "avatarUrl": "https://.../avatar.png"
    }
  }
}
```

> 前端存储：token、role、user。

### 2.2 退出登录

- **URL**：`POST /auth/logout`
- **鉴权**：是

**Response：**
```json
{ "code": 0, "msg": "ok", "data": true }
```

### 2.3 获取当前登录用户

- **鉴权**：是

**Response：**

## 3. 文件上传

### 3.1 通用上传

- **URL**：`POST /files/upload`
- **鉴权**：是
- **Content-Type**：`multipart/form-data`

**FormData：**
| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| file | File | 是 | 图片/语音等 |
| bizType | string | 否 | `tongue` / `avatar` / `feedback` / `voice` 等 |
| patientId | string | 否 | 可选，后端也可从 token 推导 |
| taskId | string | 否 | 随访任务 id |
| alertId | string | 否 | 告警 id |

**Response：**
```json
{
  "code": 0,
  "msg": "ok",
  "data": {
    "fileId": "f_90001",
    "url": "https://your-cdn.com/files/xxx.jpg",
    "name": "tongue.jpg",
    "size": 123456
  }
}
```

> 前端会使用 `data.url` 作为展示地址，并保留 `fileId` 用于随访记录关联。

## 7. SSM 实现建议（简要）

- Controller 层建议：
  - `@RestController`
  - `@RequestMapping("/api")`
- 统一返回包装：
  - `Result<T> { int code; String msg; T data; }`
- 统一异常处理：
  - `@ControllerAdvice` + `@ExceptionHandler`
- Token：
  - SpringMVC 拦截器 `HandlerInterceptor` 做鉴权 & 注入当前用户
- 上传：
  - SpringMVC `MultipartResolver`
  - 返回文件 `url`（CDN/静态资源映射）与 `fileId`

```json
{
  "code": 0,
  "msg": "ok",
  "data": true
}
```

## 6. 建议的数据表/实体（简要）

> 仅供参考，后端可按实际数据库设计。

- `user`：账号（patient/staff 统一）
- `patient_profile`：患者资料
- `patient_basic_info`：基础信息
- `daily_measurement`：每日监测记录
- `lab_result`：化验结果
- `rehab_task`：康复任务
- `tcm_survey`：中医问卷
- `message`：消息中心
- `feedback`：用户反馈
- `alert`：告警事件
- `followup_task`：随访任务
- `followup_record`：随访记录
- `file_object`：上传文件元数据（fileId/url/mime/size/bizType/...）

## 7. 联调说明（给后端同学）

1. 先实现：`/auth/login`、`/patient/home`、`/staff/tasks`、`/staff/alerts`，即可跑通最小闭环。
2. 返回结构务必统一：`{ code, msg, data }`。
3. 注意微信小程序：
   - request/upload 合法域名配置
   - HTTPS（生产）
   - 图片 url 可访问（否则预览失败）
4. 若后端暂时不做 JWT，也可用 session/cookie：
   - 需保证 `uni.request` 能携带 cookie（小程序端默认会带同域 cookie）。
